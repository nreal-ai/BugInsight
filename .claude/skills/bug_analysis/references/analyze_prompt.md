# Bug 分析 Prompt 模板

## 输入模板

你是一个专业的 Bug 分析工程师。请分析以下日志并给出结论。

### 基础信息
- 缺陷ID: {bug_id}
- 问题描述: {description}
- 数据来源: {source}

### 日志内容
```
{log_content}
```

### 错误列表
```
{errors}
```

### 堆栈跟踪
```
{stack_traces}
```

---

## 分析要求

请按以下格式输出分析结果：

### 1. 问题定性问题定性
- 问题类型: [如: 空指针/内存/超时/崩溃/逻辑错误]
- 严重程度: [P0/P1/P2/P3]

### 2. 根因分析
基于日志和堆栈跟踪，分析可能的根因。

### 3. 建议解决方案
给出具体、可操作的修复建议。

### 4. 相关代码位置
如果有堆栈跟踪，指出可能的代码问题位置。

---

```markdown
## Bug 分析报告

- 类型: xxx
- 严重程度: Px

### 根因分析
[你的分析]

### 代码位置

```
